# 日记蒸馏与归档工作流 (Daily Journal Distillation)

> 该工作流定义了 Agent 如何处理每天生成的流水账日志 (`memory/daily/*.md`)，确保短期记忆不堆积，长期价值被提纯。

## 触发条件
- 每日凌晨的心跳 (Heartbeat) 巡检。
- 或用户主动要求“整理一下这几天的日记”。

## 执行步骤

### 1. 扫描与锁定
- 定位 `memory/daily/` 目录下所有 **非今日** 的 `.md` 文件（因为今日文件还在活跃写入中）。
- 逐个读取这些历史文件。

### 2. 价值提取 (Distillation)
利用大模型的原生认知能力，审视流水账中的内容，提取以下“金线”：
- **新习得的偏好**（如：用户不喜欢用这个库、用户喜欢这种回复格式）。
- **踩坑与经验**（如：某个特定环境下的报错必须这么改）。
- **悬而未决的长期任务**。
将上述提纯内容使用 `write/edit` 追加合并到核心 `MEMORY.md` 的合适位置。如果内容与现有条目重复，则进行归并。

### 3. 降级封存 (Archival)
- 提取完毕后，原始的 `YYYY-MM-DD.md` 将失去活跃价值，但不应直接删除（防追溯）。
- 创建或确认存在降级目录：`memory/distilled/`。
- 将已处理的文件从 `memory/daily/` 移动 ( `mv` ) 到 `memory/distilled/` 目录中。

## 目标效果
- `memory/daily/` 目录保持极度清爽（通常只有 1-2 个文件）。
- 避免大量流水账被吸入主 Prompt 上下文导致 Token 浪费和注意力稀释。
